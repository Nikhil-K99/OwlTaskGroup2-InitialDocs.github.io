    IMPORTANT:

    BEFORE deploying to HEROKU, make sure to comment out the section in the pom.xml labeled "COMMENT-OUT SECTION FOR HEROKU DEPLOYMENT". This is the section that enables the typescript-generator utility for synchronizing the Java messsage data types with the TypeScript message data types. This utility is unable to run in Heroku.

    Be sure to uncomment this section during development so that the typescript-generator utility can be used to generate the TypeScript message data types.

    Note that the configuration of the typescript-generator will need to be modified to add the paths to the Java message data types being used in this project.

    In addition, the Dockerfile will need to be modified so that the proper controller class is being used.

    IAPiDefs.BALLWORLD_CONTROLLER_PACKAGES needs to include the package name of this project's controller class that is being used. Do NOT delete the Ballworld controller package as this enables Ballworld to still be used for testing purposes.

    Notes

    This project is configured as a Next.js frontend written in React using TypeScript plus a Java/Spring framework backend.
    React is designed for "single page applications" where the user interface is generated and mutated dynamically rather than navigating to completely new web pages for every action.

    The project, as cloned, is a fully operational web application that uses bidirectional, asynchronous JSON message passing via a websocket connection.

    As-cloned application features:

    • Available types to add (once the backend is operational):
      • Diagonal (pre-loaded) -- Moves in a straight diagonal direction.
      • Horizontal -- Moves in a horizontal direction.
      • Vertical -- Moves in a vertical direction.
      • Breathing -- Radius changes in a sinusoidally varying manner.
      • ColorChanging -- Changes to random colors.
      • Curving -- Moves in a curving direction.
      • Wander -- Moves in a randomly changing directions.
      • Queen -- Moves in randomly changing directions but also makes smaller Child balls that follow it around. Child balls have a random lifespan and their Queen mother ball will flash random colors when they die.
      • Student-defined types

    IMPORTANT: Configuring the project for its first use with React

    See the Using React page in Canvas for complete instructions!

    1. IntelliJ should automatically prompt you to let it run npm install as soon as the project loads.
      • If not automatically prompted, from a terminal window, run npm install
      • This will create an node_modules folder filled with many, many libraries.
    2. Right-click the src_java/main/java folder and select Mark Directory as.../Sources Root if it is not already marked as such.
    3. Likewise, make sure that the target folder is marked as Excluded.

    ===========================================================

    Compiling the project

    Both the Next.js/React frontend and the Java backend are automatically compiled.
    There is nothing that the developer needs to do.

    ===========================================================

    Running the project locally

    IMPORTANT: The application consists of TWO independent servers, the Node.js server being run by the Next.js frontend and the Spring framework server being run by the Spring framework (Spring Boot) backend. Make sure that BOTH servers are started!

    ALWAYS START THE FRONTEND FIRST then the backend as the backend assumes that the frontend is already running!

    Starting the Frontend

    The provided code should include an IntelliJ run configuration called Next.js Dev Server. From the Run/Debug Configurations menu, select the Next.js Dev Server run configuration and click the green run icon next to it.

    Stopping the Node.js development server: Click the red Stop button next to the Run/Debug Configurations menu.
    If both servers are currently running, you will be given the option to stop one or both servers.

    Starting the Backend

    Starting the Spring framework server: The provided code should include an IntelliJ run configuration called SpringAppController (Strategy). From the Run/Debug Configurations menu, select SpringAppController (Strategy) and click the green Run button next to it.

    Stopping the Spring framework server: Click the red Stop button next to the Run/Debug Configurations menu.
    If both servers are currently running, you will be given the option to stop one or both servers.

    Viewing the app: Open up a browser tab to http://localhost:8080.

    IMPORTANT:

    The Spring framework backend server MUST be restarted after ANY change to the Java code!
    (Select the TestServer run configuration and click the green Restart button.)

    The Node.js frontend server does NOT need to be restarted after editing the frontend code (be sure to save the file!) but a refresh of the browser page is generally needed to see the changes.

    The difference is that Java is compiled while JavaScript is interpreted.

    Testing just the front-end using the Node.js dev server

    This technique is for a quick test to see if the front-end is rendering properly.

    Run the Next.js Dev Server run configuration BUT open the browser to http://localhost:3000.

    This will run just the React front end using the local Node.js development server.
    Navigation to different pages should work but expect any functionality that involves the backend, e.g. websocket and REST calls, not to work properly.
